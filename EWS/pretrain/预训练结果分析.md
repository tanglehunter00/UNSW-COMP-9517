无标签样本由 12714 个 npy 文件构成，原始素材来自甜菜 2016 分割挑战赛，原始尺寸 1296×966 px，左右对称裁剪至 966×966 px 后缩放至 350×350 px，并将亮度调整至与 EWS 数据集大致相同的平均水平。

预训练时的验证集不另设数据源，而是沿用同一份 npy 路径列表，按列表顺序每隔固定条数取一条作为验证样本，默认间隔为 500，其余条目全部参与训练。每个 epoch 结束在验证集上计算重建损失，并在训练过程中保留验证损失最低时对应的权重为 best 检查点，供后续分割微调载入。

在早期实验中尝试过更规整的分割方式：将图像切分为多个 350×350 px 的切片，再转为 npy。但该流程对甜菜 2016 挑战赛数据并不理想，因为单张图幅面较大，而绿色植物往往集中在部分区域（常见为中部），切片结果中大量样本几乎只有裸土或极少量植被，预训练信号偏弱。鉴于该数据集分辨率较高，采用较简化的切块策略即可获得足够多的、含植被的样本供模型学习；即便损失部分空间上下文，信息量通常仍足够。

预训练阶段曾尝试 50%～75% 不等的遮盖比例，以及 5～25 不等的 patch size。在各类设置下训练约 20 个 epoch 后，结合验证损失与保存的检查点，对重建效果做抽样目视比较；其中在 75% 遮盖且 patch size 较大（如 25）时问题最突出：可见 token 过少，模型易退化为用大段同色块铺满画面以压低误差。

在 patch size 为 5、遮盖率 50% 的设置下，模型能学到较充分的视觉结构：训练损失单调下降，验证损失虽有波动但整体重建质量较好，绿色植物以及部分土壤、石块的外轮廓可被较合理地还原。

## English version

The unlabeled set comprises 12,714 `.npy` patches. Source images come from the Sugar Beets 2016 segmentation challenge. Frames are originally 1296×966 pixels; we apply a symmetric horizontal crop to 966×966, resize to 350×350, and adjust brightness to approximately match the mean level of the EWS dataset.

For pre-training we do not use a separate validation corpus: we keep the same path list and, in file order, take every 500th entry for validation by default while the rest are used for training. After each epoch we compute reconstruction loss on the validation split and retain the checkpoint with the lowest validation loss so far as the best model for later segmentation fine-tuning.

An early, more regular pipeline tiled each image into multiple 350×350 crops before converting to `.npy`. That approach fit the Sugar Beets 2016 data poorly: scenes are wide, while green vegetation often occupies only part of the field of view (often the center), so many tiles were nearly bare soil or contained very little foliage, weakening the pre-training signal. Because the native resolution is high, a simpler cropping scheme already supplies enough plant-rich patches; losing some global context is usually acceptable.

We explored mask ratios from 50% to 75% and patch sizes from 5 to 25. After roughly 20 epochs per configuration, we compared checkpoints using validation loss and qualitatively compared reconstructions across settings. Under 75% masking with a large patch size (e.g. 25), behavior was worst: too few visible tokens led the model toward filling the canvas with large uniform color regions to minimize error.

With patch size 5 and 50% masking, training is stable and informative: training loss decreases monotonically, validation loss is somewhat noisy but reconstruction is strong overall—green vegetation and, to a useful degree, soil and stone boundaries are recovered.
